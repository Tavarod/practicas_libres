Análisis del Problema: El Carrito de Compras Desordenado

1. Descripción del Problema

En una tienda online, los productos agregados al carrito de compras a veces aparecen listados múltiples veces en lugar de agruparse con su cantidad total si se añaden en diferentes momentos.

Objetivo: Describir la lógica para procesar y mostrar correctamente la lista de productos del carrito, agrupando los productos idénticos y sumando sus cantidades.

2. Datos de Entrada para la Lógica

La entrada es una secuencia de acciones de "agregar producto". Cada acción puede ser:

  • Simplemente el ID_Producto (asumiendo que cada vez que se "agrega", se añade 1 unidad).
  • O un par (ID_Producto, cantidad_agregada).

Ejemplo de secuencia de IDs si se agrega 1 unidad a la vez: ["prod_A", "prod_B", "prod_A", "prod_C", "prod_A", "prod_B"]

3. Lógica de la Solución Propuesta

Se utilizará un diccionario (o mapa hash) para llevar la cuenta de los productos y sus cantidades acumuladas. Llamaremos a este diccionario carrito_agrupado.

  1. Inicialización

    Crear un diccionario vacío:

    carrito_agrupado = {}

  2. Procesamiento de cada producto agregado

    Para cada producto que el usuario intente agregar (identificado por ID_actual y con una cantidad_actual, que puede ser 1 por defecto):

    • Si ID_actual ya existe como clave en carrito_agrupado:

      Obtener la cantidad previamente almacenada y sumarle la cantidad_actual.

      cantidad_existente = carrito_agrupado[ID_actual]
carrito_agrupado[ID_actual] = cantidad_existente + cantidad_actual
    • Si ID_actual NO existe como clave en carrito_agrupado:

      Agregar el ID_actual como nueva clave al diccionario, asignándole la cantidad_actual.

      carrito_agrupado[ID_actual] = cantidad_actual

  3. Resultado Final

    Una vez procesados todos los productos de la entrada, el diccionario carrito_agrupado contendrá los productos correctamente agrupados.

    Ejemplo:

    Si la entrada fue una secuencia de IDs (asumiendo cantidad 1 cada vez): "producto_123", "producto_456", "producto_123", "producto_123", "producto_789".

    El diccionario carrito_agrupado resultante sería:

    {
    "producto_123": 3,
    "producto_456": 1,
    "producto_789": 1
}

4. Resumen del Enfoque

La clave de esta solución es el uso de un diccionario donde el ID del producto actúa como clave única, y el valor asociado es la cantidad total acumulada de dicho producto.

Este enfoque es eficiente porque la búsqueda, inserción y actualización de elementos en un diccionario suelen tener un rendimiento promedio muy bueno (tiempo constante, O(1), en muchos casos).
